# i18n

Apply these rules whenever writing, reviewing, or translating any user-visible text in this project.

## Two layers

All text in this app goes through one of two layers — the wrong choice for a given surface is a bug.

| Layer | What it covers | How to use |
|---|---|---|
| **UI strings** | Static labels, errors, button text, page headings | `const { t } = useTranslation(); t('pages.catalog.heading')` |
| **CMS content** | Dynamic data coming from `cms.json` (names, descriptions, bio…) | `const l = useLocalize(); l(item.title)` |

Never mix them: don't hardcode a UI label, don't run a `LocalizedText` object through `t()`.

## Languages

Two supported languages, defined in `src/services/i18n/i18n.ts`:

```ts
export const LANGUAGES = ['fr', 'en'] as const;
```

- Default and fallback: `fr`.
- Persisted to `localStorage` under key `artisan_language`.
- A missing `t()` key falls back to `fr` silently in production; logs a `console.warn` in DEV.

## UI strings — translation files

Files: `src/assets/locales/fr.json` and `src/assets/locales/en.json`.

Key structure (nested, grouped by surface):

```
breadcrumb.*
common.*           ← shared labels (close, retry, loading)
footer.*
header.*
pages.cart.*
pages.catalog.*
pages.categories.*
pages.checkout.*
pages.cms.*        ← CMS loading/error states
pages.contact.*
pages.home.*
pages.item.*
pages.notFound.*
```

Rules:
- Always add a key to **both** locale files. The `en` file must never lag behind `fr`.
- `fr` keys are the source of truth for missing-key fallback.
- Interpolation uses `{{variable}}` syntax (e.g. `"remove": "Retirer {{title}} du panier"`).

## CMS content — `useLocalize`

`LocalizedText` is defined in `src/services/providers/cms/types.ts`:

```ts
type LocalizedText = { fr: string; en?: string };
```

`en` is always optional — the `fr` field is the required fallback. The `useLocalize()` hook resolves to `en` only when the current locale is `en` **and** an `en` value exists; otherwise it returns `fr`.

```tsx
const l = useLocalize();
// returns item.title.en if locale=en and en is set, else item.title.fr
<h1>{l(item.title)}</h1>
```

## Price formatting

Use `useFormatPrice()` from `src/services/i18n/formatPrice.ts` — never construct `Intl.NumberFormat` directly in a component.

```ts
const format = useFormatPrice();
format(item.price); // "12,50 €" or "€12.50" depending on locale
```

The hook maps `en` → `en-GB` and everything else → `fr-FR` for `Intl.NumberFormat`.

## Language switcher

`LanguageSwitcher` (in `src/layout/appDrawer/`) calls `i18n.changeLanguage(lang)` and relies on the `languageChanged` event to persist to `localStorage`. Don't duplicate that logic elsewhere.
