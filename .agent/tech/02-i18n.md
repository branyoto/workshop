- `src/i18n/types.ts`

- `src/i18n/localize.ts`
- `src/layout/LanguageSwitcher.tsx`

## Reused components

- Reuse `Button` or `IconButton` for language switcher controls.
- Reuse `Drawer` placement for the drawer-bottom language switcher.

## Explicit URLs

- Language switching does not change URLs.
- The current explicit URL remains unchanged when switching between French and English:
  - `/`
  - `/catalog`
  - `/catalog/:categoryId`
  - `/catalog/:categoryId/:subcategoryId`
  - `/catalog/:categoryId/:subcategoryId/:subId`
  - `/item/:id`
  - `/checkout`
  - `/contact`
  - `/login`
  - `/admin/cms`

## Storage

- localStorage key: `artisan_language`
- Allowed values: `fr`, `en`
- Default: `fr`

## Implementation details

- Use `i18next` with `react-i18next` for UI translations.
- Add `i18next` and `react-i18next` to dependencies.
- Configure i18next in `src/i18n/i18next.ts`.
- Use `useTranslation()` for UI labels.
- Implement `localize(value)` for CMS localized content.
- Missing English CMS content falls back to French.
- Missing translation keys should display a development-friendly placeholder such as `[missing:key]`.
- Language changes must not clear cart state, filters, or current route.
- Persist language changes in `artisan_language`.
